## Project

Emoji Nook is a native Linux emoji picker built with Tauri v2 + React 19. It runs in the background, pops up an overlay window on a global shortcut, and injects the chosen emoji into the previously focused app. The picker adapts its appearance to the host desktop via `xdg-desktop-portal`.

## Architecture

`pnpm` + Cargo workspace monorepo. See `docs/architecture.md` for diagrams and data-flow detail.

- `apps/emoji-picker/src/` â€” React 19 + TS frontend. `App.tsx` does view routing (picker vs settings) and window lifecycle; `hooks/useTheme.ts` maps portal theme info to CSS custom properties; `hooks/useSettings.ts` persists settings via tauri-plugin-store. All styling lives in `App.css` as CSS custom properties.
- `apps/emoji-picker/src-tauri/src/` â€” Rust/Tauri backend. `lib.rs` sets up plugins, tray, shortcut registration, and IPC commands (`insert_emoji`, `show_picker`, `hide_picker`, `update_shortcut`); `injection.rs` implements the clipboard-shuffle emoji injection.
- `plugins/xdg-portal/` â€” `tauri-plugin-xdg-portal` crate bridging `xdg-desktop-portal` via `ashpd` (theme info + Wayland global shortcuts). `guest-js/` holds the TS bindings consumed by the app; `dist-js/` is the pre-built output.

### Cross-cutting behaviours to keep in mind

- **Display-server split:** behaviour forks on `WAYLAND_DISPLAY`. Wayland uses the portal's GlobalShortcuts (bound once; shortcut changes need an app restart). X11 uses `tauri-plugin-global-shortcut` (can re-register live via `unregister_all` + `on_shortcut`). Keep this Wayland/X11 branching explicit.
- **Emoji injection** is a clipboard shuffle (save â†’ write emoji â†’ Ctrl+V â†’ restore), with paste simulated by `ydotool` â†’ `wtype`/`xdotool` fallback. These are runtime system dependencies, not bundled.
- **The picker window** is created once and only shown/hidden (never destroyed). It is a frameless, transparent, always-on-top overlay. Theme is re-fetched on each show via the `picker-shown` event.

### Tauri v2 / plugin specifics

- Use Tauri v2 **capabilities** (not v1 allowlists). App capabilities live in `apps/emoji-picker/src-tauri/capabilities/`.
- Plugin commands are registered in `plugins/xdg-portal/build.rs`; prefer **autogenerated** command permissions under `plugins/xdg-portal/permissions/autogenerated/commands/`. Don't add manual permission files that duplicate them, and keep `permissions/default.toml` minimal. Permissions used by the app must match real permission identifiers.
- Prefer direct imports; avoid barrel re-export files unless requested.
